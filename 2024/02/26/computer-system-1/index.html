Chapter 03 Representation of Numeric Data

3.1 Integer

Focus on binary number with N Bits ( called \({X_T}\) )

\[X_T=X_{n-1}X_{n-2}…X_1X_0\]

3.2 Encoding Methods of Integers

3.2.1 Original code

use \(X_{n-1}\) as signature

0 owns two representations — +0 & -0

3.2.2 Complement code ( IMPORTANT )

Eg in 12 o’clock

\(10 - 4 = 6\)

\(10 + 8 = 18 \equiv 6 ( \mathrm{mod}\ 12 )\)

Definition \(\forall X_T: [X_T]_C=M+X_T(\mathrm{mod}\ M)\)

The Conversion from Complement code to true value:

  • If the sign bit is 0 / the number is positive: The value part does not need to be changed
  • Else the sign bit is 1 / the number is negative: Invert the bits of value part, and add 1 to the result

From \([X_T]_C\) to \([-X_T]_C\): Invert every bits, and add 1 to to the result

Advantages ( compared with sign-magnitude ):

  • No +0 or -0 anymore
  • Hardware simplicity

3.2.3 Inverse code

Complexity and uselessness in modern computer

Difference with the complement code Just invert, no add

\(-1\rightarrow -0\rightarrow+0\rightarrow 1\)

Chapter 05 Floating-Point Number Representation & Non-Numeric Data

5.1 IEEE Standard 754

Numerical Form: \(\mathrm{(-1)^sM\cdot2^E}\)

  • Sign bit s determines positive or negative
  • M is in range [1.0, 2.0)
  • Exponent E weights value by power 2

 

5.1.1 Precision

  • Single precision 32 bits: s=1; exp = 8; frac = 23
  • Double precision 64 bits: s = 1; exp = 11; frac = 52
  • Extended precison 80 bits (Intel only)

 

5.1.2 Exponent (Frameshift Code)

Before calculation, times each number by \(2^{n-2}\) (or frameshift by \((n-2)\) bits), and divide the same number after calculation

the goal is to escape from the case of “deviding zero” and etc.

When turn the component number to the true number, \(E=exp-bias\)

 

5.1.3 Normalized Encoding

Value: \(15213.0_1\kern{}_0=1.1101101101101_2\times2^{13}\)

Significand - M = \(1.1101101101101_2\) - frac = \(1101101101101000…0_2\)

Exponent - E = \(13\) - Bias = \(127(=2^{8-1})\) - Exp = \(140\) = \(10001100_2\)

Result - \(0 | 10001100 | 1101101101101000...0\)

 

5.1.4 Denormalized Values & Encoding

In normalized value range, number lower than \(1.0\times 2^{-126}\) is missing and undefined

In denormalized value range, numbers in than such range definates different meanings

Additionly, M in denormalized values will be regarded as \(0.100…_2\) but not \(1.100…_2\), and meanwhile the Exp is \(000…0_2\) which means \(\times2(^{-(n-2)})_1\kern{}_0\)

normalized value cannot represent ZERO without denormalized version

  1. \(exp = 000…0; frac = 000…0\): Zero
  2. \(exp = 000…0; frac \ne 000…0\): Numbers close to zero
  3. \(exp = 111…1; frac = 000…0\): INF (infinity)
  4. \(exp = 111…1; frac \ne 000…0\): NaN (Not-a-Number)

1 & 2 show denormalized values whie 3 & 4 show another two special values

 

5.1.5 Special Properties

  • FP Zero same as Integer Zero
  • Can (Almost) Use Unsigned Integer Comparison
    • Must consider \(0 = -0\)
    • Must first compare sign bits
    • NaNs problematic

7.1.3 Theorems: n Variables

De Morgans’s Theorem

\[\overline{F(X_1,X_2,…+X_n)}=F(\overline{X_1},\overline{X_2},…,\overline{X_n}),(+\ \mathrm{or}\ \cdot)\]

Shannon’s Theorem

\[F(X_1,X_2,…,X_n)=X_1\cdot F(1,X_2,…,X_n)+\overline{X_1}\cdot F(0,X_2,…,X_n)\]

\[F(X_1,X_2,…,X_n)=[X_1+F(0,X_2,…,X_n)]\cdot[\overline{X_1}+ F(1,X_2,…,X_n)]\]
