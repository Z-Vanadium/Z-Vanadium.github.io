堆：堆操作的本质是维护（二）
2023/12/6

0. 择优而取 | 堆的建立

在上一篇的结尾，我们留下了一个问题，就是如何建立一个堆。

在建立一个堆之前，我们需要数据。假设我们已经有了N个数据，储存在数组Array里面，接下来我们的任务就是利用Array的数据建立一个堆Heap。

回忆上一篇文章，我们主要讨论了堆的插入与删除。而为了实现这两个操作，我们专门分别设计了两个维护函数，分别是向上和向下维护的函数，而我们建立一个堆的过程，恰好就是一个破坏-维护的过程。

为什么这么说，首先假定我们有了一个拥有N-1个节点/元素的堆Heap，我们要插入一个数据，获得一个N个节点的堆，那我们插入的新数据，不正是破坏么，而我们接下来要做的就是维护这个被破坏的堆。最后重复这样的操作N次，我们就可以得到一个合法的大堆。

接下来我们要书写代码了么？

先等等，我们要知道，维护一个堆是有两种方法的，分别向上和向下维护，那么显而易见的是，建立一个堆应该也有两种方法。

首先考虑向上维护来建立一个堆。

如果要向上维护，我们要做的很简单，对于Array里面的每一个数，我们只需要将其插入Heap当中就可以了。

回顾一下原先的Insert函数代码。

int Insert(MaxHeap MaxH, ElementType x){
    if(IsFull(MaxH))    return ERROR;
    MaxH->MaxSize+=1;

    int child=MaxH->MaxSize;
    ElementType* array=MaxH->Array;

    array[child]=x; //x入堆
    return UpShift(MaxH, child);
}

我们唯一要做的事情就是遍历数组Array，对其中所有元素都进行一次插入操作，这个堆就建立好了。

我们考虑其时间复杂度，对于第 i 个数据，我们要进行比较交换的次数为 logi 次，因此时间复杂度为 O(nlogn) ，这个时间复杂度已经很优秀了。

接下来考虑向下维护。

如果我们要向下维护一个堆，我们一定要保证这个堆的子堆是合法的堆。如果我们一个数据一个数据插入，我们能保证子堆，也就是最底下的堆是合法的堆吗？

显然不行。因为我们插入到堆底的数据是任意的，不一定就小于其父堆。

因此我们考虑从最后一个非叶节点开始，向上维护堆。

下面给出代码。

MaxHeap ArrayToMaxHeap(ElementType* array, int N){  //通过array建立MaxHeap，返回建立的MaxHeap
    MaxHeap MaxH = BuildMaxHeap();
    MaxH->MaxSize = N;
    int i;
    for(i=0;i<N;++i)    MaxH->Array[i+1]=array[i];
    for(i=N/2;i>=1;--i)    DownShift(MaxH, i);
    return MaxH;
}

首先我们建立一个空堆MaxH，大小比Array大1（注意到MaxSize指向的是MaxH下标的最后一个，而不是MaxH的大小）。

接下来我们无视堆的规则，直接将Array存入堆，这样一个堆是一个无序的堆。

然后我们从堆的 N/2 处开始向下维护，（不从N处开始是为了剪枝，因为N/2+1 ~ N处的节点都没有子堆），如此维护到根节点，一个堆就建立好了。

考虑其时间复杂度，可以发现对于数组Array，我们进行了 1 次遍历和 N/2 次向下维护，而向上维护的复杂度为 O(logn) ，根据数学知识，我们可以计算得出向下维护建堆的复杂度为 O(N) 。

为什么是logn而不是nlogn？参见这篇文章（需要一些数学知识，因此这里Vanadium略去不写）
可以看到，向下维护建堆的方法优于向上建堆（插入建堆），因此我们择优而取，选择向下维护的方法建堆。

1. 优秀的原地排序算法 | 堆排序

所以……有了一个堆，我们能做什么呢？

接下来，我们就可以做很多事情了！

比方说，堆排序。

排序算法大家应该不陌生，在C程课上，我们学习过冒泡排序、选择排序、交换排序等一系列 O(N2) 排序算法，也有接触过高效的快速排序等排序算法。

然而在我们有了堆的概念之后，我们是否可以做一个全新的排序算法呢？

我们知道，堆的特点就是根节点的值永远是最大（小）值，所以我们只需要重复删除堆的节点，并把删除的节点依次输出即可。

所以我们的问题就是，如何建立一个堆，巧的是，这个问题刚才我们已经解决过了，就是向下维护建堆。

我们可以轻松的给出代码。

void HeapSort(ElementType* array, int N){   //对array原地进行堆排序
    MaxHeap MaxH = ArrayToMaxHeap(array, N);
    int i;
    for(i=0;i<N;++i)   array[i]=Delete(MaxH);
    return;
}

可以看到，我们在解决了建堆、删除两个操作之后，实现堆排序的算法不要太简单。

不过有细心的朋友可能会注意到Vanadium所说的“原地排序”，但是代码实现里面根本不是原地排序，因为额外申请了一个Heap的空间。

实际上这是因为，我们的代码是实例化一个ADT，是需要封装的，如果我们只是应用堆的思想，而不拘泥于堆的格式，我们可以直接利用Array进行排序——毕竟Array和Heap的array明显是一样的。

具体的代码实现就不书写了。

HeapSort的时间复杂度也是 O(nlong)，而空间复杂度为 O(1) ，并且效率基本稳定。因此在一些容易MLE（虽然很少）的大量数据的题中，HeapSort会体现出它的优势。

堆：堆操作的本质是维护（一）
2023/12/6

0. 完全二叉树与堆 | 堆的储存与实例化

上回书说到，我们可以用数组版的完全二叉树实现一个堆，在C语言中，我们可以方便地（没有封装的类你说什么方便？）用结构体Struct来实现某个ADT（因此建议读者在学习结构体之后再来品读以下文字）。

#define ElementType int
#define MAXN 105
#define ERROR -1

typedef struct MaxHeapRoot* MaxHeap;
struct MaxHeapRoot
{
    ElementType* Array;
    int MaxSize;    //指向Array中最后一个元素的下标
};

其中Array属性储存了一个实现完全二叉树的数组的指针，MaxSize属性储存了数组中最后一个元素的下标。

由于我们利用的是完全二叉树，我们可以把完全二叉树的知识迁移过来：

在数组储存的完全二叉树中，任一节点 Node[i] 的两个孩子节点满足 Left_Child = Node[2*i] 且 Right_Child = Node[2*i+1] ，其父亲节点满足 Parent = Node[i/2]

这里需要注意的是，下标i是从1而不是0开始的，并且访问父亲/儿子时可能会出现越界的情况，这些是我们在实例化的时候要注意的。

与此同时，由于堆的递归定义得知的，我们可以注意到，一个堆的左右子树也是堆。因此，一个合法的堆，应该满足对于任意i∈ [1, MaxSize] ， Node[i] > Node[2*i] ，Node[i] > Node[2*i+1]。

我知道有人不喜欢看公式，那我就用自然语言改写一下：对于一个合法的大根堆来说，它满足任意一个以i为下标的节点的值，要大于以2i和以2i+1为下标的节点的值。

接下来我们要考虑实例化的函数了，我们需要做的事情是：（1）分配内存给Array；（2）初始化MaxSize的值；（3）还有……？

MaxHeap BuildMaxHeap(){
    MaxHeap MaxH=(MaxHeap)malloc(sizeof(struct MaxHeapRoot));
    MaxH->Array=(ElementType*)malloc(sizeof(ElementType)*MAXN);
    MaxH->Array[0]=INT_MAX; //初始化哨兵
    MaxH->MaxSize=0;
    return MaxH;
}

可以看到我在完成了上述前两个步骤之外，还做了一件事：给 Array[0] 赋了一个INT_MAX(代表int类型的最大值2^32-1)，并给了他一个名字：哨兵。

哨兵这个概念你可能之前未曾接触过，不要着急，就当是作者在这里留下的一块小糖，等到时机成熟，自然会甜甜的（？）。

至此，我们可以通过BuildMaxHeap函数获取一个指针heap指向一个大根堆了，我们实现堆的长征完成了第一步。

1. 模块化思维 | 维护堆的函数（向上）

所以……有了一个堆，我们能做什么呢？

先来点最简单的吧，我们要判断一个堆究竟是空的还是满的。

这一步的意义在于：我们增删的过程中，一定要考虑堆的空间是否足以支持我们的操作，否则会导致越界，产生奇怪的小错误。

int IsEmpty(MaxHeap MaxH){
    return MaxH->MaxSize==0? 1:0;
}
int IsFull(MaxHeap MaxH){
    return MaxH->MaxSize==MAXN-1? 1:0;
}

IsEmpty和IsFull的实现实在是太简单了，我们进入下一步吧！

按照Vanadium的要求，我们要实现的功能是插入（增加）数据和删除（取出）数据两个功能。先别急着书写Insert和Delete函数，我们要想想，在我们插入或删除的时候，堆究竟发生了什么变化。

首先考虑插入一个元素，由于数组的插入只能在末尾进行，因此，我们先将元素Item作为叶子节点放入最后一个没有两个儿子节点的父节点下面。

放入之后，与AVL树类似，我们要考虑改变的节点有没有破坏什么。在AVL树里面，改变某个节点会破坏原有的平衡因子；而在堆里面，改变某个节点（可能）会破坏其以父节点为根的一个堆的合法性。类似的，我们也要进行维护。

由于在插入元素的时候，我们首先破坏的是最下层的堆的合法性，所以我们需要一个向上维护的函数UpShift。

我们要如何设计一个UpShift函数呢？考虑其实现方式。

首先我们需要确定UpShift所需要的参数，一个MaxHeap是显然的，由于我们维护某个堆的时候，可能会破坏其父堆的合法性，因此我们要进行递归维护父堆的合法性，所以我们需要一个参数t确定我们应该从何处开始向上维护。

综上所述，我们可以这么定义UpShift：

int UpShift(MaxHeap MaxH, int t){    
    /*递归边界 | 判断是否需要继续维护t的父堆*/
    /*维护t的父堆的代码*/
    UpShift(MaxH, t/2);
    return 0;
}

然而，就Vanadium本人来说，他是特别不喜欢递归调用的，因为递归调用经常会爆栈而且速度慢，所以他会使用While来替代递归调用。

如果我们考虑while的退出条件，不难想到，当t指向的位置不合法的时候，也就是t<=1的时候，就不需要维护了。

除此以外还有一种情况：如果 Node[t] 的改变没有破坏父堆的合法性，换言之，父堆仍然满足一个合法堆的条件——父堆的值依然大于以2*t和以2*t+1为下标的节点的值——那么也就不需要继续维护了。

下面请看代码。

int UpShift(MaxHeap MaxH, int t){  //从t开始，向上调整堆
    if(t>MaxH->MaxSize || t<1) return ERROR;
    int child=t;
    int parent=child/2;
    ElementType* array=MaxH->Array;
    while(1){
        if(array[child]>array[parent]){
            ElementType temp;
            temp=array[child];
            array[child]=array[parent];
            array[parent]=temp; //交换父子结点
            child=parent;   parent=child/2; //向上递归
        }
        else    break;
    }
    return 0;
}

代码的实现和我们的想法如出一辙有一点点不一样。聪明的你应该发现了，Vanadium貌似没有判断边界条件，也就是我们认可的 “当t指向的位置不合法的时候，也就是t<=1的时候，就不需要维护了” 。

如果Vanadium只是简单的忘记了，那么这段代码肯定会出问题，因为当我维护到child = 1 时，接下来我们会判断 parent = child/2 = 0……等等，还记得我们之前对 Array[0] 做了什么么？

是的，我们给它赋了一个值，一个最大最大的INT值，在这种情况下，以它为根的堆，一定是合法的大根堆，不是么？

它的存在就相当于一位哨兵，所有操作遇到他时候统统要停下来。

有了UpShift函数，我们就可以保证插入之后，堆依然能保持自己的纯洁性合法性了。

因此我们可以快速写出Insert函数：

int Insert(MaxHeap MaxH, ElementType x){
    if(IsFull(MaxH))    return ERROR;
    MaxH->MaxSize+=1;
    int child=MaxH->MaxSize;
    ElementType* array=MaxH->Array;
    array[child]=x; //x入堆
    return UpShift(MaxH, child);
}

2. 模块化思维 | 维护堆的函数（向下）

接下来让我们快马加鞭地写出Delete函数吧！

由于Vanadium的要求，我们所要取出的值一定是最大值，也就是堆的根，因此取值是很简单的，返回堆的根，也就是Array[1] 的值就可以了；但可惜的是，失去了堆的根，堆还能是堆吗？显然不是了。

所以，我们需要将堆的最后一个叶子上的值赋给堆根，并删除这个叶子，这样，堆至少还是一个完全二叉树，不至于失了自己的根。

同样的，我们要考虑进行维护了。

与刚才不一样，我们删除某个堆的子节点，堆依旧是合法的，但是当我们把某个值赋给堆的根的时候，堆的合法性就难说了，因此我们现在要做的事情是维护堆的根，然后向下维护，DownShift。

考虑向上维护的函数使用了递归，向下维护的函数自然也是需要递归的。然而不同于子节点只有一个父节点，一个父节点通常有两个子节点，但是假如我们把两个子节点一一判断过去，代码就会很冗长马，失去美观。

这里我们不加证明地给出向下维护的代码。

int DownShift(MaxHeap MaxH, int t){  //从t开始，向下调整堆
    if(t>MaxH->MaxSize || t<1) return ERROR;    
    int parent = t;  //parent为待处理节点，child为左孩子
    int child = 2*parent;
    int maxsize = MaxH->MaxSize;
    ElementType* array=MaxH->Array;    
    while(child <= maxsize){    //当左孩子存在时
        if(child+1 <= maxsize && array[child+1] > array[child]){   //当右孩子存在，且大于左孩子
            ++child;
        }   //此时child指向parent两个（如果有）子节点中的较大者
        if(array[child] > array[parent]){   //当较大的孩子大于父节点
            ElementType temp;
            temp=array[child];
            array[child]=array[parent];
            array[parent]=temp; //交换父子结点
            parent = child;
            child = 2*parent;
        }
        else    break;
    }
}

可以看到，Vanadium使用了一个很取巧的办法同时实现了判断两个子节点的存在性和大小比较。

首先，对于parent，先取它的左儿子 child = 2*parent。如果左儿子就不存在，右儿子一定不存在，此时说明parent是叶子节点，可以直接结束递归；如果左儿子存在，我们直接进行判断右儿子是否存在以及和左儿子的大小比较。

两个操作在一个if语句中完成后，child会指向左右儿子中较大的一个（如果只有左儿子，自然指向左儿子）。

最后，再比较这个较大的儿子和父节点，如果父节点大于其中较大的儿子，说明这个堆是合法的，就可以结束递归了；反之则递归维护以较大的儿子作为根的堆。

上面这个方法可能有些抽象，建议读者多读两遍，自行画图分析一下！

理解并实现了DownShift函数后，我们就可以很快的给出Delete函数的代码了。

ElementType Delete(MaxHeap MaxH){
    if(IsEmpty(MaxH))    return ERROR;
    ElementType* array = MaxH->Array;
    ElementType res = array[1];
    array[1]=array[MaxH->MaxSize];
    MaxH->MaxSize-=1;
    DownShift(MaxH, 1);
    return res;
}

数据结构杂谈（一）
2023/12/6

这一篇文章本来应该作为数据结构的第0节的

鲁迅先生曾经说过：“是什么？为什么？怎么办？这是一个问题”

1. ​什么是数据结构？

开篇立论，定义先行。但是可惜的是，关于“数据结构”，鲜有一个标准化的定义。参照多方定义，我在这里给出我的一个定义，不一定对，但有可能错：数据结构是数据及其关系和对其的特定操作——算法的集合。

是的，很抽象。

那我就用人话说一遍：假设胡图图、胡英俊、张小丽是三个数据，那么，父子、母子、夫妻就是关系，张小丽打胡图图屁股，胡图图去找胡英俊求助，就是特定操作了。

如果用计算机语言表达，C语言里的数组Array就很显然属于是数据结构，属于数据之间的关系，而你去试图去查找，增删里面的特定元素就是特定操作了。

话还没说完，如果你对计算机组成原理有一定的了解，你会知道，数据在计算机中是以010101的二进制储存在内存条（也可能是CPU的寄存器、缓存等等，关于这个请不要杠我，竑嬸除外）里面的。

数组也不例外，在物理层面上，的确，Array[0]正是储存在Array[1]前面，以此类推，一个一个排排队。换言之，数组元素的地址是连续的。

因此，我们容易猜到，遍历数组的时候，我们只需要命令计算机挨家挨户，一个一个一个地址找过去就可以了。

上述的数据储存方式，或者说，数据结构，是很直观的，然而，不幸的是，生活中绝大多数数据结构是没办法按照简简单单的Array储存的。比如上述胡图图一家，你能给我想个办法储存在计算机内部，表示出父子、母子，还能给我表示出打屁股、求助这些操作吗？很难的啦~

因此，为了研究更普遍的情况，聪明的计算机科学家想到了抽象数据类型（ADT）。所谓抽象数据类型，不就是我上面说的数据结构加个抽象么？那所谓抽象，就是不具体——不依赖计算机内部物理储存（我才不管地址连不连续，我只管逻辑关系，好比不管胡图图在翻斗花园还是幼儿园，他都是胡英俊的儿子），不依赖某种特定编程语言。

而我们接下来，在数据结构里面讨论的所有所有东西，都将是ADT。虽然我会用C语言给出代码实例，但是你可以将代码背后的「思想」用任何一门编程语言赋予到你的计算机中，任何，任何。

2. 为什么要数据结构？

​为了更优的算法。

算法工程师的终极目标就是解决某个特定问题，数据结构存在的终极目标就是为了让算法工程师能设计一个更优的算法。

那什么是「优」呢？我们知道，光速具有时空不变性，因此，我们可以从时空两个角度评判一个算法的优劣（这有什么关系哇呀呀呀呀！）

学术上用时空复杂度描述一个算法的优劣，说白了就是建立了一个输入数据规模增长速度与算法执行过程的时空消耗增长速度之间的函数。

什么意思呢？我们来想一个很呆呆的具体问题，比方说我们将胡图图一家人（包括爷爷奶奶等）看作数据，要求查找到胡图图的爷爷。

根据ADT的思想，我们可以设计很多数据结构，正常人应该会把胡图图一家人组成「族谱」吧，这样只需要从胡图图开始，往上搜索，很快就能找到他爷爷；

但是我偏不，我要将胡图图一家人按照左手第二根手指的长度排序，那么，从胡图图家祖孙十八代中试图搜寻到他的爷爷，绝对是一件麻烦事。

而且，当我想去寻找他奶奶，他外婆等等，前者一定比后者更优。

反过来想，如果我想应用胡图图-胡英俊-胡爷爷这个浅显易懂的查找路径查找到胡爷爷，前者家谱般的储存结构必不可少。

综上所述，优秀的数据结构与优秀的算法总是双生的，这也就是我们学习数据结构的根本目的。

堆：高效的优先队列（零）
2023/12/6

0. 从优先队列…

Vanadium开了一家小店，专门贩卖自己的爱，可是没想到，开业第一天就陆陆续续地来了超出他想象多得多的客人（而且只有熟人）排在店面外面。为了保持自己的风度，Vanadium决定把自己的爱贩卖给值得的人，需要你设计一个数据结构，既能实现客人的加入，也能按照Vanadium对每个人的好感度，每次取出队伍中好感度最高的客人。

如果我们提炼出背景中的重要信息，我们就知道：题目要求我们设计一个数据结构，它类似于队列，但是出队的顺序却是按照数据的优先级顺序。

这样的数据结构，我们称之为优先队列（PriorityQueue）。

欸等等等等，我们不是在讲堆吗，怎么扯到了优先队列了？

没事，等我说完什么是堆，你就知道为什么我愿意花这么多篇幅讲优先队列了。

刚刚我们说的【优先队列】，还只是停留在ADT阶段，接下来我们要考虑实例化这个ADT。按照要求，我们需要实现的功能是（1）储存数据；（2）插入数据；（3）删除优先级最高的数据。

来，PK。

【数组】如果我们考虑数组，插入是简单的，只要在数组末尾放入数据即可，但是如果我们要寻找到优先级最高的数据，我们就需要先遍历一遍数组，再删除其中某个数据，删除时，我们需要将该数据后的每一个数据依次往前移一位（不然数组不就断断续续了么）

【链表】相较于数组，链表在删除数据上有着优势，因为它只需要遍历一次，然后稍稍修改指针，就完成了删除，然而时间复杂度依然不低

【有序表】对于有序数组/链表，时间复杂度大头就在维护上面了，依然不是我们的最优选择。

所以，我们只能止步于此了么？

1. 什么是堆？我们怎么实现堆？

当然不是！

二叉搜索树！

相较于传统数据结构，二叉搜索树有着搜索快（O(logn)），易于维护等等优点，那我们就试着用二叉搜索树实现我们高效的优先队列吧。

考虑到Vanadium每次要取出的是好感度最高的客人，我们可以将好感度最高的客人安排在根节点，以保证我只要访问根节点就可以取出该客人；

考虑到树的递归定义，我们只要保证每个父节点的值都大于其两个子节点，我们的树就可以保证根节点的值是最大的了；

考虑到普通的二叉树会有很多的冗余（比如某棵只有右子树的树，你不就是个线性表吗哎呦喂），我们应该保证我们的树应当是一棵完全二叉树！

考虑到完全二叉树是完全的，是可以用数组表示的，我们可以简单的用数组储存我们的树，这样做的好处还有一个就是，我们可以轻松的通过子节点访问父节点。

考虑完这么多要求，我们的，全新的实现方式就产生了：利用数组储存一棵完全二叉树，保证其中每个父节点的值都大于其两个子节点。这就是堆（Heap）。上面的堆，我们称之为大根堆（MaxHeap）；同样的，如果根节点是最小值，那我们称之为小根堆（MinHeap）。

其实还蛮形象的，堆就是一堆东西，下面是小件物品，支撑着上面的大件物品（bushi）。

2. 储存堆可不是我们的终极目标

回想起我们杂谈中谈到的，数据结构不单单包括数据之间的关系（储存数据），更重要的是算法——我们可以做什么。

对于堆，由于我们创造它的目的是实现优先队列，因此，我们仅考虑有意义的操作（无意义操作举例：交换两个子节点>.<）：

首先是优先队列的取出操作：对于一个堆，我们要去查找其中最大值是很容易的，根节点就是最大值。但是之后的事情就使我们犯难了，因为我们要维护这个堆，总不能一取，这个堆就失去灵魂了吧？

其次是优先队列的插入操作：如果只是一棵普通的自平衡二叉查找树（AVL树），我们只需要简单的插入一个节点，然后轻松的根据破坏者和被破坏者的关系选择RR/RL/LR/LL旋转就好了。但是我们的堆是借助数组实现的，数组树能简单的旋转吗，旋转后还会是完全二叉树吗？

可以看到，处理一个普普通通的增删问题，就已经使我们汗流浃背了。辣末，我们要如何实例化一个堆呢，按下不表，且听下回分解。
