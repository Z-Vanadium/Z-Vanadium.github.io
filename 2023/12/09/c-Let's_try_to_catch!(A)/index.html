Let's try to try-catch! (A)

2023/12/9 C/C++

背景

宏

宏是C语言提供的一种实用的编译预处理功能之一，通过带参数的宏定义和不带参数的宏定义，可以用一个 指定的标志符来进行简单的字符串替换或者进行阐述替换。形式如下：

#define PI 3.1415926535
#define MAX(a, b) ({ __typeof__ (a) _a = (a); __typeof__ (b) _b = (b); _a > _b ? _a : _b; }) // This only works on gcc

以上的两个例子分别实现了把代码中的PI标识符替换为3.1415926535的宏定义，以及一个可以用来计算两数最大值的MAX宏定义。

异常

像C++/Java/Python等等高级编程语言都有异常机制：当一个函数遇到自身无法处理的错误时，它可以选择抛出（throw）这个“异常”，让异常逐级向上层传播，直到遇到上层某个函数用catch捕获这个异常对它进行处理，处理完毕后在被捕获的位置继续程序；或者如果没有能处理这个异常，那么程序就会崩溃掉。这在调用函数嵌套层级很多，而你想要把一个错误传播到上层时很有用：不用占用函数返回值的位置返回错误码了，也免得修改中间不知道多少函数的函数签名。

下面是一个C++里的例子：

void do_something() {
    printf("Before Exception\n");
    throw std::exception("Error Message");
    printf("Unreachable\n");
}
void func_114() { do_something(); }
// ...
void func_1() { func_114(); }
int main() {
    try {
        func_1();
    } catch (std::exception const &e) {
        printf("%s\n", e.what());
    }
    printf("Resumed...\n");
}
// OUTPUT:
// Before Exception
// Error Message
// Resumed...

这个例子里面，在do_something()函数中我们抛出了一个异常。程序在此中止，一路向上层寻找有没有可以捕获的函数，最后找到了main()函数。main()函数捕获这个异常，并在捕获结束后在捕获的位置继续程序的运行。

异常很好很强大，唯一的问题是，C语言并不支持异常。

emmm，问题不大，那我们自己来实现一个吧~

必要知识点

实现try-catch机制的关键是如何实现跨函数的跳转，你可能会想到使用goto. 但遗憾的是goto只能在函数内部跳转，所以我们需要更强大的跳转能力——setjmp和longjmp

#include <setjmp.h>
int main(void) {
    jmp_buf env;
    int i = setjmp(env); // 记录当前位置，setjmp返回0
    printf("i = %d\n", i);
    if (i != 0) exit(0); // 第一次经过时i==0，第二次经过时i==2

    longjmp(env, 2); // 控制流回到setjmp被调用的地方，然后i被赋值为新的返回值2
    printf("Unreachable\n"); // 到达不了这里
}

setjmp和longjmp不必在同一个函数中也能实现跳转。例如，下面的代码就实现了从g()跳转回f()

#include <setjmp.h>
static jmp_buf env;
void f( void ) {
    int x = setjmp( env );
    if (x != 0) return; // avoid infinite loop
    g();
}
void g( void ) { longjmp( env, 1 ); }

PS: 其实有限制，需要setjmp的调用者还在函数调用堆栈上，所以你不能把setjmp包装一层再拿来用。

活用setjmp/longjmp和宏来在C语言中实现try-catch机制吧！

目标和要求

本题目标语法如下面的裁判程序所示，要求：

  • 实现类似C++的基本try-catch语法结构

  • catch_(e)会在catch语句块中创建一个名字为e的变量，它的值等于被throw_抛出的值

  • 支持嵌套的try语句块（包括被调用函数里的try_语句块导致的间接嵌套）
    本题的一些其他假定：

  • catch_(e)的变量e总是int类型，throw_(x)抛出的x总是一个int类型的值

  • 每个throw_都会被catch_，所以不必处理没catch_程序退出的情况

裁判程序示例

你需要使下面的程序通过编译并输出正确结果。

#include <stdio.h>

/* Do your magic here~ */

void foo(void) { throw_(114514); }

int main(void) {
    try_ {
        try_ {
            printf("Hello\n");
            foo();
            printf("Unreachable\n");
        } catch_(e) {
            printf("Inner Exception: %d\n", e);
            throw_(-e); // rethrow
        }
    } catch_(e) { 
        printf("Outer Exception: %d\n", e); 
    }
    return 0;
}

Outer Exception: -114514
