C++ | 模板

发表于2023-12-16|更新于2024-03-14
|阅读量:

前言

在OOP中, 我们经常遇到要用到一些程序片段, 他们实现和完成的功能基本相同, 不同的只是数据类型, 此时我们可以利用模板template大大简化程序书写, 提高简洁性和可读性.

 

正文

函数模板

template <class 类型参数...>   //模板声明头
返回值 函数名(形参表){
    函数体
}

//定义一个求最大值的函数模板
template <class T>
T MaxElement(T array[], int size){
    T _max = array[0];
    for(int i=1;i<size;++i)
        if(array[i]>_max)
            _max = array[i];
    return _max;
}

 

隐式调用

当我们像使用普通函数一样直接使用模板函数时, 就会进行隐式调用, 编译器会自动为我们匹配合适的重载

例如, 当我们运行以下测试程序

int main(){
    double a[3]={1.5, 2.9, 2.1};
    cout<<MaxElement(a,3)<<endl;

    int b[4]={5, 2, 6, 4};
    cout<<MaxElement(b,4)<<endl;
}
/*输出结果:
2.9
6
*/

在行3和行6中, MaxElement分别被实例化为浮点型和整型函数, 将鼠标悬停在函数上面可以看到

 

显式调用

此外, 我们也可以在调用时直接声明类型参数

int main(){
    double a[3]={1.5, 2.9, 2.1};
    cout<<MaxElement<int>(a,3)<<endl;   //Error
}

此时声明了整型的MaxElement就会报错, 因为a是浮点型数组

 

重载

模板函数可以重载, 只要他们的形参表或类型参数表不同, 以下重载都是合法的

template <class T>
void func(T x, T y){
}

template <class T1, class T2>
void func(T1 x, T2 y){
}

template <class T1, class T2>
void func(T1 x, T1 y){
}

 

重载的匹配顺序

当存在同名普通函数和模板函数重载时, 编译器的匹配顺序如下

  1. 参数表完全匹配的普通函数(指非模板实例化的函数)

  2. 参数表完全匹配的模板函数

  3. 参数表通过自动类型转换后能够匹配的普通函数

  4. 报错

注意: 编译器不会进行自动类型转换以匹配隐式调用的模板函数

template <class T>
void func(T x, T y){
    cout<<x<<' '<<y<<endl;
}

int main(){
    int x=9;
    double y=8.1;
    func(x,y);      //Error
    func<int>(x,y); //Accepted
}

 

类模板

template <class 类型参数...>   //模板声明头
class 类名{
    成员
};

类模板名<真实类型参数表> 对象名(构造函数参数表);

//定义一个键值对类
template <class T1, class T2>
class JsonNode{
    public:
        T1 key;
        T2 value;
        JsonNode(T1 _key, T2 _value):key(_key), value(_value){};
        bool operator<(const JsonNode<T1,T2> & tmp) const;
        bool operator>(const JsonNode<T1,T2> & tmp) const;
};

template <class T1, class T2>   //注意这里也要声明是模板函数
bool JsonNode<T1,T2>::operator<(const JsonNode<T1,T2> & tmp) const{
    return this->value < tmp.value;
}

template <class T1, class T2>   //注意这里也要声明是模板函数
bool JsonNode<T1,T2>::operator>(const JsonNode<T1,T2> & tmp) const{
    return !(*this<tmp);
}

int main(){
    JsonNode<string,int> A("Mike",10);
    JsonNode<string,int> B("John",15);
    cout<<(A<B)<<endl<<B.key;
}
/*
输出结果:
1
John
*/

注意: 同一个模板生成的不同类, 例如JsonNode<string,int>和JsonNode<string,double>是不兼容的

 

非类型参数

函数模板和类模板的类型参数表都能接受非类型参数(但要求是类型typename, 而不是类class), 以下用法是合法(但不常用)的

template <class T, int size>
T MaxElement(T array[]){
    T _max = array[0];
    for(int i=1;i<size;++i)
        if(array[i]>_max)
            _max = array[i];
    return _max;
}

int main(){
    int array[4] = {2,6,5,9};
    cout<<MaxElement<int,4>(array)<<endl;   //这里通过模板生成了只能接受大小为4数组的函数
}

以及

template <class T, int size>
class Array{
    private:
        T array[size]={0};
    public:
        void Print(){
            for(int i=0;i<size;++i) cout<<array[i]<<'#';
            cout<<endl;
        }
};

int main(){
    Array<int,10> int_array;
    int_array.Print();
}

 

类模板的派生

  • 类模板从类模板派生

  • 类模板从模板类派生

  • 类模板从普通类派生

  • 普通类从模板类派生

注意主语和限定词:

  • 类模板指的是一个尚未实例化的模板, 被继承时仍然是模板

  • 模板类指的是一个已经实例化的类, 本质上与普通的类是一样的

类模板从类模板派生

template<class T1, class T2>
class A{
    public:
        T1 v1;
        T2 v2;
};

template<class T1, class T2>
class B:public A<T2,T1>{
    public:
        T1 v3;
        T2 v4;
};

template<class T>
class C:public B<T,T>{
    public:
        T v5;
};


int main(){
    B<int,double> obj1; //v1~v4: double, int, int, double
    C<char> obj2;       //v1~v5: char, char, char, char, char
}

类模板从模板类派生 | 类模板从普通类派生

当类模板从模板类或普通类派生时, 由这个类模板生成的类均会以这个模板类或普通类为基类

template<class T1, class T2>
class A{
    public:
        T1 v1;
        T2 v2;
};

template<class T1, class T2>
class B:public A<int,char>{
    public:
        T1 v3;
        T2 v4;
};

class C{
    public:
        int v5;
};

template<class T3, class T4>
class D:public C{
    public:
        T3 v6;
        T4 v7;
}


int main(){
    B<int,double> obj1; //v1~v4: int, char, int, double
    D<char,double> obj2;       //v5~v7: int, char, double
    D<short,float> obj2;       //v5~v7: int, short  , float
}

普通类从模板类派生

除语法外, 与普通的类的派生无异

template<class T1, class T2>
class A{
    public:
        T1 v1;
        T2 v2;
};

class B:public A<int,char>{
    public:
        T1 v3;
        T2 v4;
};

 

模板与友元

函数、类、类的成员函数作为类模板的友元

void func(){}

class A{};

class B{
    public:
        void g(){}
}

template <class T>
class C{
    friend void func();
    friend class A;
    friend void B::g();
}
//所有由C实例化的类, 均有以上三个友元

函数模板作为类模板的友元

template <class T1, class T2>
class A{
    public:
        template <class T1, class T2>
        friend void func(){};
};
//所有函数模板的实例化, 都是类模板实例化的友元

类模板作为类模板的友元

template <class T1, class T2>
class A{
    public:
        template <class T1, class T2>
        friend class B{};
};
//所有B类模板的实例化, 都是A类模板实例化的友元

 

类模板与静态成员变量

如果类模板中定义了静态成员变量, 则其所有实例化共用一个静态成员变量
