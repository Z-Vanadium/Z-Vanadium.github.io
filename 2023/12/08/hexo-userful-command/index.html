常用Hexo命令归档

2023/12/8 Hexo 前端

记录一些常用的Hexo命令/配置方法


1. 部署相关

summary

撰写前

hexo g -w

撰写后

hexo d && hexo s

调整CSS/ejs文档（F5自动更新）

hexo s

将本地仓库生成展示页并且配置到GitHub（需要提前建立ssh链接）

hexo c && hexo g && hexo d

检查ssh链接

ssh -T -v git@github.com

在本地仓库生成展示页并且配置到本地服务器localhost:4000
一般用于预览

hexo c && hexo g && hexo s

修改端口

hexo server -p [port]

修改IP

hexo server -i [IP]

监视文件变化并动态生成展示页面

hexo generate --watch
hexo g -w

2. 新建相关


新建文章/页面

hexo new [layout] [file_name]

布局layout:
默认值为post
可选值

layout path
post source/_posts
page source
draft source/-drafts

将草稿转为文章

hexo publish [layout] [file_name]

使用模板新建

hexo new [scaffold] [file_name]

此时Hexo会尝试在/scaffold文件夹中寻找scaffold.md，并根据其内容新建文章


3. 写作相关


使用<--! more -->以分隔文章的摘要以及正文
或者在Front-matter中设置description字段
eg.

...
description: |
    [markdown]
    [markdown]
...

注意：| 和缩进是格式的一部分，不能省略


图片援引
参考文章

本地引用
在/source里新建文件夹images，将图片统一放到/source/images中

![](images/image_name.png)

网络引用
需要图床

![image_name](image_link)
<!-- following is useful by using html tags, too -->
<img src="image_link">
<!-- pay attention to write SRC, not SCR -->

内部引用
将图片编码后储存在md文件后，直接调用

![avatar][base64str]
[base64str]:data:image/png;base64,decoded_image_by_base64

base64编码方式 | 利用python脚本  三体人语言就是NB

import base64
#base64 to image
bs='iVBORw0KGgoAAAANSUhEUg....' # 太长了省略
imgdata=base64.b64decode(bs)
file=open('2.jpg','wb')
file.write(imgdata)
file.close()

#image to base64
f=open('723.png','rb') #二进制方式打开
ls_f=base64.b64encode(f.read()) #读取文件内容，转换为base64编码
f.close()
print(ls_f)
